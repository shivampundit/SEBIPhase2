## üí° Study Tips

### For MCQ Exams

1. **Read carefully**: Pay attention to keywords like "MUST", "CAN", "CANNOT"
2. **Eliminate wrong answers**: Use process of elimination
3. **Language-specific**: Know which language the question is about
4. **Common traps**: 
   - Abstract class can have constructor ‚úÖ
   - Interface can have default methods (Java 8+, C# 8+) ‚úÖ
   - Multiple inheritance in C++ ‚úÖ, Java ‚ùå
   - Python uses name mangling, not true encapsulation

### For Coding Questions

1. **Start with the abstract**: Identify interfaces/abstract classes first
2. **Apply SOLID**: Check if your design violates any principle
3. **Consider edge cases**: Null checks, empty collections, inheritance chains
4. **Memory management**: Especially important in C++
5. **Thread safety**: Consider concurrent access

### Common Mistakes to Avoid

‚ùå **Don't**:
- Try to instantiate abstract classes
- Forget to implement all abstract methods
- Mix up method overloading vs overriding
- Ignore access modifiers
- Create deep inheritance hierarchies (> 3 levels)
- Use inheritance when composition is better

‚úÖ **Do**:
- Use interfaces for contracts
- Keep classes focused (SRP)
- Prefer composition over inheritance
- Document abstract methods clearly
- Use virtual destructors in C++ base classes
- Validate in setters, not getters
